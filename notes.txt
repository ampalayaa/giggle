Digital Product E-Commerce System

Database Schema (possible)
-- users table
id (PK), firstname, middlename (nullable), lastname, email, password, role, status, last_login, created_at

-- product table
id (PK), title, description, price, acategory, file_path, thumbnail, is_active, download_limit, created_at

-- orders table
id (PK), user_id (FK), total_amount, payment_status, payment_method, created_at

-- order_items table
id (PK), order_id (FK), product_id (FK), price

-- transactions table
id (PK), order_id (FK), user_id (FK), transaction_type (Purchase, Refund, Cancel), amount, status, reference_number, remarks, created_at

-- audit_trail table
id (PK), user_id (FK), action, target_table, target_id, timestamp


Admin - Full control: manage users, products, reports
Staff - can process transactions, vieww reports
Client - can register, order, download product

🖼️ Recommended Design Flow:
Design index.php first (product grid preview).

Then product_detail.php (single product layout with Download / Buy Now button).

Then cart.php & checkout.php.

Once the customer side looks good, start the admin area.

Connect to the database when frontend is locked.
